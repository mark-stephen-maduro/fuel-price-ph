# Fuel Price Monitoring PH

Fuel Price Monitoring PH is a DOE-based fuel information web app for the Philippines.

Its purpose is to make official fuel monitoring data easier to read. Instead of pretending to show live pump prices, it presents the latest available DOE-monitored regional snapshot, week-over-week movement, regional comparisons, and recent trend movement in a simple interface.

## What This Is For

This app is for users who want a quick reference for:

- latest DOE-monitored fuel prices
- weekly fuel price adjustments
- regional fuel price averages
- short-term fuel price trends

This app is not for:

- real-time station pricing
- exact brand-level pump prices
- location-by-location live fuel lookup

The goal is clarity and trust. The product is intentionally positioned around official published data, with a clear disclaimer that actual prices may vary.

## How It Works

The current version is a static React app that renders DOE-derived JSON files into dashboard sections.

### Current flow

1. The frontend starts in [main.jsx](/Users/mark/Business/fuel-price-ph/src/main.jsx).
2. [App.jsx](/Users/mark/Business/fuel-price-ph/src/App.jsx) loads the app shell and page sections.
3. [useFuelData.js](/Users/mark/Business/fuel-price-ph/src/hooks/useFuelData.js) fetches the fuel datasets when the app mounts.
4. [fuelDataApi.js](/Users/mark/Business/fuel-price-ph/src/services/fuelDataApi.js) reads:
   - [latest.json](/Users/mark/Business/fuel-price-ph/public/data/latest.json)
   - [history.json](/Users/mark/Business/fuel-price-ph/public/data/history.json)
5. Those files are generated by [sync-doe-data.mjs](/Users/mark/Business/fuel-price-ph/scripts/sync-doe-data.mjs), which:
   - resolves current official DOE `prod-cms` PDF URLs
   - downloads official DOE regional PDF files
   - parses the PDFs into normalized JSON
   - computes week-over-week movement from consecutive official DOE publications
6. [fuelData.js](/Users/mark/Business/fuel-price-ph/src/utils/fuelData.js) derives the dashboard view model, such as:
   - focus region
   - highest gasoline region
   - normalized dashboard state
7. Presentational components render the sections:
   - [HeroSection.jsx](/Users/mark/Business/fuel-price-ph/src/components/HeroSection.jsx)
   - [LatestPricesSection.jsx](/Users/mark/Business/fuel-price-ph/src/components/LatestPricesSection.jsx)
   - [CompareSection.jsx](/Users/mark/Business/fuel-price-ph/src/components/CompareSection.jsx)
   - [TrendsSection.jsx](/Users/mark/Business/fuel-price-ph/src/components/TrendsSection.jsx)
   - [MethodologySection.jsx](/Users/mark/Business/fuel-price-ph/src/components/MethodologySection.jsx)

### In plain terms

The app does not fetch live market data from a backend. It reads normalized JSON generated from official DOE files and turns that into a readable fuel dashboard.

That means:

- the frontend is fast
- deployment is simple
- GitHub Pages is enough
- the DOE sync script can update the data files without rewriting the UI

## What The User Sees

The app currently has five main sections:

- `Home`: explains the product and shows the latest publication date
- `Latest Prices`: shows current DOE-monitored fuel averages and week-over-week movement
- `Compare`: compares the latest accessible regional averages and shows the source week for each row
- `Trends`: shows recent weekly adjustment movement
- `Methodology`: explains scope, limitations, and planned automation

## Data Model

The app uses two generated JSON files in `public/data/`.

### `latest.json`

This file contains:

- source metadata
- effective date
- last updated timestamp
- coverage note describing the latest accessible official archive date
- week-over-week movement values for the focus region
- regional fuel price ranges, averages, and source PDF links

### `history.json`

This file contains:

- a list of recent NCR week-over-week movement entries
- one record per effective date
- product-specific movement for gasoline, diesel, and kerosene

## Why The Architecture Looks Like This

The current architecture is:

`DOE prod-cms PDFs -> parser -> normalized JSON -> static frontend`

That architecture fits this use case because:

- DOE data is expected to be published on a schedule, not streamed in real time
- GitHub Pages can host the UI cheaply and simply
- JSON snapshots are easy to validate and inspect
- the frontend can stay independent from scraping logic
- the PDF archive can be re-synced from a single script

## Official Data Source

The app currently syncs from official DOE `prod-cms` PDFs.

Examples of current official source URLs:

- NCR: `https://prod-cms.doe.gov.ph/documents/d/guest/ncr-price-monitoring-03102026n-pdf`
- Mindanao: `https://prod-cms.doe.gov.ph/documents/d/guest/10-lfro-price-monitoring-march-10-2026-pdf`

Important date note:

- As of March 17, 2026, the current sync resolves official DOE files dated March 10-16, 2026 for NCR and Mindanao.
- Region coverage is intentionally limited to sources with stable, fetchable 2026 URL patterns.
- The app shows the exact DOE week and source PDF link in the UI.

## Local Development

Requirements:

- Node.js 20+
- npm

Install dependencies:

```bash
npm install
```

Start the dev server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build:

```bash
npm run preview
```

Sync the DOE data:

```bash
npm run sync:doe-data
```

## Automation

Nightly automation is configured in [.github/workflows/sync-doe-data.yml](/Users/mark/Business/fuel-price-ph/.github/workflows/sync-doe-data.yml).

- Schedule: `0 16 * * *`
- Equivalent local time: `12:00 AM` in `Asia/Manila`
- The workflow runs the DOE sync, verifies the app build, and commits updated JSON if the source data changed

## How To Update The Data

Normally, updating the displayed fuel data means running:

```bash
npm run sync:doe-data
```

That command regenerates:

- [latest.json](/Users/mark/Business/fuel-price-ph/public/data/latest.json)
- [history.json](/Users/mark/Business/fuel-price-ph/public/data/history.json)

If DOE changes the current `prod-cms` document naming pattern, [sync-doe-data.mjs](/Users/mark/Business/fuel-price-ph/scripts/sync-doe-data.mjs) is the file to update.
